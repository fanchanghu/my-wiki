# 投机解码

---

## 一句话定义
> 用小模型快速“猜”出 $K$ 个候选 token，再用大模型的**单次**前向过程进行并行验证，从而在不改变输出分布的前提下加速 LLM 推理。

---

## 核心思想

| 模型 | 角色 | 特点 |
|-----|------|------|
| **Draft 模型** | 快速生成候选序列 | 小、快、可能不准 |
| **Target 模型** | 验证候选序列 | 大、慢、权威 |

流程：

1. Draft 模型一次性生成 $K$ 个候选 token
2. Target 模型一次前向，计算这 $K$ 个位置的真实分布 $p$
3. 按顺序逐个验证，找到第一个被拒绝的位置
4. 从该位置按修正分布重新采样，后续 token 全部作废
5. 若全部 $K$ 个 token 都被接受，则 target 前向得到的第 $K+1$ 个位置分布 $p_{K+1}$ 可直接采样一个 **奖励词元（Bonus Token）**，无需额外前向

---

## 采样与验证规则

对（从 draft 分布 $q$ 中采出的）第 $i$ 个候选 token $x_i$：

**接受概率**：

$$
P(\text{接受 } x_i) = \min\left(1,\ \frac{p(x_i)}{q(x_i)}\right)
$$

**若被拒绝**，从 residual 分布采样：

$$
x \sim p'(x) = norm(\max(0,\ p(x) - q(x)))
$$

### 正确性

将目标分布拆成两部分：

$$
p(x) = \underbrace{\min(p(x), q(x))}_{\text{交叠部分}} + \underbrace{\max(0,\ p(x) - q(x))}_{\text{残留部分}}
$$

- 接受路径贡献 $\min(p,q)$
- 拒绝路径贡献 $\max(0, p-q)$
- 两者加权混合严格恢复 $p$

---

## 关键性质

| 性质 | 说明 |
|-----|------|
| **分布不变** | 投机解码输出  ≡  直接用 target 模型采样输出 |
| **加速效果** | 取决于 draft 模型与 target 模型的匹配程度，越匹配加速效果越好 |
| **并行验证** | target 一次前向验证 $K$ 个 token |
| **顺序接受** | 第一个被拒绝位置之后的 token 全部作废 |

---

## 效率分析

设每次 draft 生成 $K$ 个 token，平均接受 $\alpha$ 个：

- **有效加速比** ≈ $\dfrac{\alpha + 1}{1 + \text{draft 相对开销}}$
- 理想情况下若全部接受，一次 target 前向换 $K$ 步
- 实际中 $\alpha$ 取决于 draft 与 target 的分布相似度

影响效率的因素：

| 因素 | 影响 |
|-----|------|
| draft 模型质量 | 越接近 target，接受率越高 |
| $K$ 大小 | 太大导致后续 token 浪费，太小加速有限 |
| 任务难度 | 高熵任务 draft 更容易猜错 |

---

## 相关概念

- **Rejection 采样**：投机解码的理论基础
- **Residual 采样**：拒绝后修正分布的关键操作
- **KV Cache**：投机解码需要为 draft 和 target 分别管理缓存
- **草稿模型（Draft Model）**：通常比 target 小 10~100 倍

---

## 一句话总结

**投机解码 = 小模型出草稿 + 大模型批审批 + 拒绝处残差修正**。它用分布式等价换取推理加速，是 LLM 服务中**兼顾正确性与效率**的重要技术。
